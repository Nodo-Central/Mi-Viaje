# Contributing

Thank you for considering a contribution to Mi Viaje. This project touches public transport, civic infrastructure, public-interest data, and user privacy, so we keep the contribution rules explicit.

## Contribution Rights

By opening a pull request, submitting a patch, filing an issue with proposed content, or otherwise intentionally sending a contribution to this repository, you represent that you have the right to submit it and license it under Apache-2.0.

You retain ownership of your contribution and may continue to use it elsewhere.

Unless explicitly agreed in writing, contributions are provided without warranty or liability, to the maximum extent permitted by law.

## What Counts as a Contribution

A contribution includes code, documentation, translations, designs, configuration, tests, issues, pull requests, suggestions, examples, and any other material intentionally submitted to this project.

## Safety and Privacy

Do not submit real card identifiers, personal data, private transit records, access keys, internal documents, confidential material, or logs that contain information you are not allowed to share.

If you are reporting a bug that involves sensitive data, describe the behavior without attaching raw dumps or screenshots that expose identifiers.

## Development Expectations

- Keep NFC reading, parsing, data storage, and UI concerns separated where possible.
- Prefer small pull requests with clear context.
- Include tests or manual verification notes when practical.
- Run `./gradlew assembleDebug` before submitting larger changes when possible.
- Do not add dependencies lightly; explain why a new dependency is needed.
